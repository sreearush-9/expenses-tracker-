System Flow

  1. User registration via /auth/register
  2. User login via /auth/login (JWT issued)
  3. Frontend stores JWT in localStorage
  4. Dashboard calls protected /expenses APIs with Bearer token
  5. CRUD operations: create, fetch, update, delete expenses
  6. Token validation in JwtAuthenticationFilter for each request
